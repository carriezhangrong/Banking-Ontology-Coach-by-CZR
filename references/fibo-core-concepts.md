# 常用 FIBO 概念小抄（对标时优先在这里找现成的）

这是银行建模最常用到的一小撮 FIBO 核心概念，便于学员"复用优先"时快速命中。
**重要：下面只给概念名、所属域、释义和检索线索；精确 IRI 一律以官方 FIBO Viewer（spec.edmcouncil.org/fibo）为准，不要凭记忆填写 IRI。**

释义为便于教学的转述，非 FIBO 官方原文措辞。

## FND 基础域（Foundations）

- **Party（当事方）**：能在各种情境中扮演角色的实体，可以是自然人或组织。是"角色"模式的根基。
- **PartyInRole（当事方角色）**：把"某当事方在某情境/合约中扮演某角色"显式建模的模式。**对手方、借款人、发行人、受益所有人都应建成 PartyInRole，而不是独立类型。**
- **Agreement / Contract（协议 / 合约）**：当事方之间达成、产生权利义务的约定。注意它与"金融工具"是两回事。
- **Relation / Date 等通用件**：日期、通用关系等基础构件，多数模型都会用到。

## BE 业务实体域（Business Entities）

- **LegalEntity（法人实体）**：具有法律人格、能独立承担权利义务、订立合约的组织。KYC 里的"客户主体"通常落在这里。
- **Ownership & Control（所有权与控制）**：股权、控制关系的建模。**受益所有权链、穿透识别就建在这一族概念上。**
- **BeneficialOwner（受益所有人）**：按"角色"理解——某自然人通过所有权/控制链对某法人实体享有最终权益，用角色模式表达。

## FBC 金融商业域（Financial Business and Commerce）

- **FinancialInstitution / Intermediary（金融机构 / 中介）**：银行、券商等市场参与机构。
- **FinancialInstrument（金融工具）**：代表金融价值或债权债务关系的工具（注意：是合约所指向/产生的标的，与协议本身区分）。
- **Account（账户）**：金融账户相关概念。

## 其余按需的域
- **SEC 证券 / DER 衍生品 / LOAN 贷款 / IND 指数指标 / CAE 公司行为 / BP 业务流程**：涉及具体工具或流程时再去对应域检索。

## 用法
对标某概念时，先按"它像哪个域"翻到上面对应段落看有没有现成的；命中就在文档的"FIBO 出处"列写明：复用/特化/自有扩展 + 域 + 概念名，IRI 留待去 viewer 核准。命中不了再回 `fibo-reference.md` 的"查→复用→特化→扩展"四步流程处理。
