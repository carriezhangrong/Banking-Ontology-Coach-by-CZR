# 新手建模常见错误清单（点评学员产物时对照检查）

发现问题时，按"**先讲为什么错 → 给判据 → 引导他自己改**"处理，不要直接替他改好。
分两组：先看银行/FIBO 特有的高频坑，再看通用坑。

## A. 银行 / FIBO 特有（优先盯这几个）

### A1. 重造 FIBO 已有概念（本领域头号反模式）
- 症状：把"法人实体""对手方""金融工具""协议"等当成全新概念自己定义。
- 后果：自断与行业标准、监管报送、跨系统集成的接口。
- 处理：带学员回 fibo-reference 走"查→复用→特化→扩展"四步，确认 FIBO 是否已有；有就复用并标注出处。

### A2. 把"角色"当成"类型"
- 症状：把"对手方""借款人""发行人""受益所有人"立成独立的顶层类。
- 判据：它是某人/某机构的**内在类别**，还是它在**某份合约/情境里扮演的角色**？是角色 → 不该当类型。
- 处理：用 FIBO 的 PartyInRole 模式——当事方（Party）通过"在某合约中担任某角色"来获得"对手方"身份。

### A3. 自有扩展不挂靠、不标注
- 症状：FIBO 没覆盖的概念，自建后既没 is-a 到 FIBO 相近父类下，也没标"自有扩展"。
- 后果：模型脱标，日后治理和对齐困难。
- 处理：尽量特化（子类化）FIBO 已有类；确实独立的，显式标注"自有扩展"。

### A4. 合约/协议 与 金融工具 混淆
- 症状：把"贷款协议"和"贷款这一金融工具"当成一个东西。
- 判据：它是当事方之间的**约定**，还是约定所指向/产生的**标的**？分属不同概念。

### A5. 概念脱离监管/治理落点
- 症状：建了一堆概念，却说不清它服务哪条胜任力问题/监管需求。
- 处理：回阶段 0，逐个问"回答哪个 CQ 需要它？"不需要的先放下。

## B. 通用坑

### B1. 把实例当成类
- 例：某具体法人"汇丰银行（北京分行）"应是 `法人实体` 类的实例，而非一个类。
- 判据：还能再分出"不同的它"吗？不能 → 是实例。

### B2. 把 part-of 当成 is-a
- 例：`分行 is-a 银行` ❌（分行不是一种银行，是银行的组成部分）→ `分行 part-of 银行` ✓。
- 判据："凡是 X 都是一种 Y"读着通顺才是 is-a。

### B3. 本该独立成类的东西被压成属性值
- 例：把"开户机构"写成 `账户.机构名称（文本）` ❌；机构本身有属性、有关系（且 FIBO 已有），应独立成类用对象属性连接 ✓。

### B4. 层级出现循环或自我包含
- 判据：is-a 必须单向、无环。沿层级走一圈找环并打断。

### B5. 同级类抽象层次不一致
- 例：`账户` 下并列着 `储蓄账户` 和某个具体账户实例——颗粒度不一致。

### B6. 定义模糊或循环 / B7. 属性缺 domain/range
- 判据：拿边界案例去套定义能否明确判定；每个属性是否都说清了属于哪类、指向什么。
